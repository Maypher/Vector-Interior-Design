Nodo admin Graphql API

Here are all the endpoints available to the admin backend of the Nodo website. It includes queries with access to private data and mutations to edit the data.

API Endpoints
https://127.168.0.8:8080/obras/graphql

Queries

ambiente

Description

Returns the ambiente matching the given ID or null.

Response

Returns an Ambiente

Arguments
Name Description
id - Int! The ID of ambiente to get.

Example

Query
query ambiente($id: Int!) {
  ambiente(id: $id) {
    id
    name
    description
    index
    obra {
      ...ObraFragment
    }
    images {
      ...ImageFragment
    }
  }
}
Variables
{"id": 987}
Response
{
  "data": {
    "ambiente": {
      "id": 123,
      "name": "xyz789",
      "description": "abc123",
      "index": 987.65,
      "obra": Obra,
      "images": [Image]
    }
  }
}

Queries

obras

Description

Returns all the obras ordered by index limited by page_size. Can be filtered by name.

Response

Returns an ObraResult!

Arguments
Name Description
page - Int! The page to get. Default = 1
pageSize - Int! The size of each page. Default = 10
name - String Filter obras by name. Default = null

Example

Query
query obras(
  $page: Int!,
  $pageSize: Int!,
  $name: String
) {
  obras(
    page: $page,
    pageSize: $pageSize,
    name: $name
  ) {
    page
    pageCount
    obras {
      ...ObraFragment
    }
  }
}
Variables
{"page": 1, "pageSize": 10, "name": null}
Response
{
  "data": {
    "obras": {
      "page": 987,
      "pageCount": 123,
      "obras": [Obra]
    }
  }
}

Mutations

createAmbiente

Description

Creates a new ambiente.

Response

Returns an AmbienteObraNotFoundAmbiente!

Arguments
Name Description
obraId - Int! The ID of the obra this ambiente will belong to.
name - String! The name of the new ambiente.
description - String The description of the new ambiente in markdown format. Can be left blank.. Default = null

Example

Query
mutation createAmbiente(
  $obraId: Int!,
  $name: String!,
  $description: String
) {
  createAmbiente(
    obraId: $obraId,
    name: $name,
    description: $description
  ) {
    ... on Ambiente {
      ...AmbienteFragment
    }
    ... on ObraNotFoundAmbiente {
      ...ObraNotFoundAmbienteFragment
    }
  }
}
Variables
{
  "obraId": 123,
  "name": "xyz789",
  "description": null
}
Response
{"data": {"createAmbiente": Ambiente}}

createImage

Description

Creates a a new image for the given ambiente. Returns an error if the ambiente wasn't found or the filetype isn't supported.

Response

Returns an ImageAmbienteNotFoundImageUnsupportedFileType!

Arguments
Name Description
ambienteId - Int! The ID of the ambiente this image will belong to.
image - Upload! The image file to upload.
altText - String! The alt text of the image. This is meant for impaired vision, not a description.

Example

Query
mutation createImage(
  $ambienteId: Int!,
  $image: Upload!,
  $altText: String!
) {
  createImage(
    ambienteId: $ambienteId,
    image: $image,
    altText: $altText
  ) {
    ... on Image {
      ...ImageFragment
    }
    ... on AmbienteNotFoundImage {
      ...AmbienteNotFoundImageFragment
    }
    ... on UnsupportedFileType {
      ...UnsupportedFileTypeFragment
    }
  }
}
Variables
{
  "ambienteId": 123,
  "image": Upload,
  "altText": "abc123"
}
Response
{"data": {"createImage": Image}}

updateImageConfig

Description

Updates an mainImageConfig to be displayed in the sites main page.

Response

Returns a MainImageConfig

Arguments
Name Description
id - Int! The ID of the imageConfig to update.
imageBorders - BordersInput What borders should the image have. Default = null
description - String The description of the image in Spanish. Default = null
descriptionEn - String The description of the image in English. Default = null
descriptionFont - String The font that should be used for the description. Default = null
descriptionAlignment - String The alignment of the description. Default = null
descriptionFontSize - Float The size used for the description in rem units. Default = null
logoPos - Direction The position of where to put the logo relative to the image.
logoBorders - BordersInput What borders should the logo have. Default = null
descriptionPos - Direction The position of where to put the description relative to the image.
overflow - Boolean Determines if the image should overflow to the edge of the screen or not.
index - Int The new zero based index of where to put the image in the main page. Default = null

Example

Query
mutation updateImageConfig(
  $id: Int!,
  $imageBorders: BordersInput,
  $description: String,
  $descriptionEn: String,
  $descriptionFont: String,
  $descriptionAlignment: String,
  $descriptionFontSize: Float,
  $logoPos: Direction,
  $logoBorders: BordersInput,
  $descriptionPos: Direction,
  $overflow: Boolean,
  $index: Int
) {

    id: $id,

    descriptionFontSize: $descriptionFontSize,

    descriptionPos

    logoPos
    logoBorders {

Types

Ambiente

Description

An ambiente is a section of an obra. It contains images.

Fields
Field Name Description
id - Int! The ID of the ambiente in the database.
name - String! The name of the ambiente.
description - String The description of the ambiente in markdown format. Can be null.
index - Float! The index of the ambiente for UI ordering purposes. It's a float due to how the database handles reordering.
obra - Obra! The obra this ambiente belongs to.
images - [Image!]! The images of this ambiente.
Example
{
